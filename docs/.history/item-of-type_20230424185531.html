Using CSS to add content

The book showed a great trick using td:nth-of-type(5):before { content: "Mac â€œ) , which would put the word "Mac" in front of the 5th <td> in the table.

Don't.

It's super tempting. Let's say you want to label all your links to outside sites. You can use this handy code:

<style>
a.external:after {
content: " (external site) ";
}
</style>

Do the link HTML like this:

<a href="https://getbootstrap.com" class="external">

You'll get this:

  • Positioning

  • Bootstrap demos

  • Bootstrap HQ

That is so awesome!

So, what's the problem? It looks perfect!

And that's the problem. It looks perfect. However, you have used CSS (which is for display) to add content. Accessibility software for blind people ignores CSS, because it's just display information that's worthless if you can't see.

This means you have two different user experiences, and that sighted people are getting important information (you're leaving the safety of our site and getting sent out into the wilds of the Internet and we no longer control what you will see). That's actually an accessibility violation; disabled users need the same information as everyone else. The CSS has content in it, and they can't access that. It's not just a decoration.

That really sucks, because I would very much like to be able to use this.
